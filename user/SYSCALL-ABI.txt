ChaserOS 用户态系统调用 ABI（调用约定 + 核心接口）
================================================

本文档定义 **ring 3 用户程序** 与 **内核** 之间通过 **`int $0x80`** 的系统调用界面。
编号与寄存器含义的权威头文件：**kernel/syscall_abi.h**  
Slime 常量模块：**integrations/slime-for-chaseros/std/chaseros/constants.sm**  
汇编桩：**user/chaseros-rt/chaseros_syscalls.asm**

--------------------------------------------------------------------
0. CPL 与 ISR 权限
--------------------------------------------------------------------

  * **用户系统调用**：必须在 **ring 3**（**CS.RPL == 3**）执行 **int $0x80**；内核在 **isr.c** 中校验 **`(regs->cs & 3) == 3`**。
  * **内核态误触发**：若非 ring 3，打印诊断信息并向调用者 **RAX** 写入 **-EPERM**（调用约定与 ring 3 一致，便于调试）。
  * **未知调用号**：**RAX** = **-ENOSYS**。

--------------------------------------------------------------------
1. 触发方式
--------------------------------------------------------------------

  * **指令**：`int imm8`，当前 **`imm8 = 0x80`**（与 **kernel/syscall_abi.h** 中 **CHASEROS_SYSCALL_VECTOR** 一致）。
  * **特权级**：仅在 **CPL = 3**（用户代码段）发起的 **int 128** 陷入按本 ABI 处理；内核态发起的同类向量由 **isr.c** 忽略或另行策略。
  * **与 Linux 的区别**：不用 **`syscall/sysret`**；参数用 **64 位寄存器** 传递，而不是 int 0x80 传统的 **ebx/ecx/edx** 布局。

--------------------------------------------------------------------
2. 寄存器调用约定（核心）
--------------------------------------------------------------------

  **调用号**

    RAX ← **CHASEROS_SYS_***（见下表）。汇编桩通常写 **EAX**，必须保证与 **RAX** 低 32 位一致。

  **参数（最多 6 个整数/指针类参数）**

    第 1 参数：**RDI**  
    第 2 参数：**RSI**  
    第 3 参数：**RDX**  
    第 4 参数：**R10**（预留）  
    第 5 参数：**R8**（预留）  
    第 6 参数：**R9**（预留）

    该顺序与 **System V AMD64** 用户态 **extern "C"** 调用约定一致，便于 **Slime `extern "C"`** 与 **`user/chaseros-rt/chaseros_syscalls.asm`** 汇编桩共用符号。

  **返回值**

    默认存放在 **RAX**。约定：**非负** 表示成功时的结果（如写入字节数）；**负值** 表示 **-errno**（错误码语义与 POSIX errno 对齐，具体在每个系统调用说明中列出）。  
    **当前内核实现**对部分调用尚未写回 **RAX**，应用勿依赖未文档化的返回值，直至对应条目标注「已实现」。

  **易失寄存器**

    内核路径可能破坏用户可见的调用约定之外的寄存器；用户侧应视 **RAX、RCX、R11** 等为易失（与多数 OS syscall 惯例一致）。若需跨 **int $0x80** 保留其它寄存器，调用方自行保存。

--------------------------------------------------------------------
3. 系统调用表（核心接口）
--------------------------------------------------------------------

  编号 **0–255** 由 ChaserOS 内核与本文档保留分配；新增调用须同步：

    * **kernel/isr.c**（分发）
    * **kernel/syscall_abi.h**（**CHASEROS_SYS_***）
    * **user/SYSCALL-ABI.txt**（本表）
    * **integrations/slime-for-chaseros/std/chaseros/constants.sm**
    * **user/chaseros-rt/chaseros_syscalls.asm**（若有裸导出符号）

  ----
  **CHASEROS_SYS_EXIT（0）**

    **用途**：终止当前用户任务并返回内核 Shell（当前实现跳转至内核提供的退出路径）。

    **参数**：**RDI** = 退出状态码（约定为 **32 位非负或按 POSIX**；当前内核可不解释）。

    **返回值**：无（不回到用户态）。**RDI** 可为退出码（内核可选用）。

  ----
  **CHASEROS_SYS_WRITE（1）**

    **用途**：向打开的文件描述符写入缓冲区。

    **参数**：  
      **RDI** = **fd**（**fd == 1** stdout、**fd == 2** stderr，均镜像控制台）  
      **RSI** = 缓冲区 **用户空间** 虚拟地址  
      **RDX** = 写入长度（字节）

    **返回值**：**RAX** = 实际写入字节数（成功）；**0** 表示 **len==0**。  
    失败时 **RAX** = **-errno**：**-EBADF**（fd 非 1 且非 2）、**-EINVAL**（**len** 超过 **CHASEROS_SYSCALL_MAX_IO_BYTES**）、**-EFAULT**（缓冲区不在用户可读映射内或长度溢出）、**-ENOSYS**（保留给其他错误路径）。  

    **权限**：缓冲区 **[RSI, RSI+RDX)** 必须通过页表校验为 **PAGE_PRESENT | PAGE_USER**（参见 **vmm_user_range_readable**），防止将 **仅内核可映射** 的 VA 当作用户缓冲区误读。

  ----
  **CHASEROS_SYS_GETPID（2）**

    **用途**：返回当前用户任务 PID（嵌入式 hello 固定为 **2**，由内核 **chaseros_active_user_pid** 提供）。

    **参数**：无（忽略 **RDI** 等）。

    **返回值**：**RAX** = PID（非负）。

  ----
  **CHASEROS_SYS_READ（3）**

    **用途**：从打开的文件描述符读入缓冲区。

    **参数**：**RDI** = **fd**；**RSI** = 用户缓冲区 VA；**RDX** = 最大读取长度。

    **行为**：  
      **fd == 0**（stdin）：无输入设备时返回 **0**（EOF）。  
      **fd == 1 或 2**：**-EBADF**（stdout/stderr 只写）。  
      **fd ≥ 3**：由 **`open`** 打开的 VFS 文件（内核记录规范化路径与读游标；含 **ext2** 与 **`/dev`** 伪节点）；读到文件尾后返回 **0**。  
    **-EINVAL** / **-EFAULT** / **-EIO**：越界长度、缓冲区不可写、底层读失败等。

  ----
  **CHASEROS_SYS_UPTIME_TICKS（4）**

    **用途**：单调 tick 计数（每次 **IRQ0** 时钟中断递增），用于简单计时/演示。

    **参数**：无。

    **返回值**：**RAX** = **chaseros_kernel_ticks**（自启动以来递增）。

  ----
  **CHASEROS_SYS_OPEN（5）**

    **用途**：在已挂载 VFS 上打开文件。**绝对路径** 以 **`/`** 开头；否则相对 **当前工作目录**（与 Shell **`cd`** 同步，跳入用户程序时从 **`shell_get_cwd`** 快照）。支持 **ext2** 卷上路径及内核 **`/dev/null`**、**`/dev/zero`** 等。

    **参数**：**RDI** = 路径 **C 字符串** 的用户 VA（以 `\\0` 结尾）；**RSI** = flags（预留，只读打开传 **0**）。

    **返回值**：**RAX** = **fd**（≥ **3**）；**-EINVAL**（VFS 未挂载等）、**-ENOENT**、**-EMFILE**、**-EFAULT**。

    **混合阶段 3**：默认构建下打开请求经 **IPC** 委派至内核 **PID=3** 服务再调用 **`user_fd_open_kpath`**（与直接 VFS 调用结果一致，便于迁出文件服务）。

  ----
  **CHASEROS_SYS_CLOSE（6）**

    **用途**：关闭 **fd ≥ 3** 由 **open** 获得的描述符。

    **参数**：**RDI** = fd。

    **返回值**：**RAX** = **0**；**-EBADF**。

  ----
  **混合内核 IPC（7–10）**

  * **CHASEROS_SYS_IPC_SEND（7）**：**RDI** = 目标 PID；**RSI** = **`message_t`** 用户 VA；成功 **RAX=0**；失败 **-EINVAL** / **-EFAULT**。
  * **CHASEROS_SYS_IPC_RECV（8）**：**RDI** = 期望发送方 PID（**0** = 任意）；**RSI** = 输出 **`message_t`**；无可用消息时 **阻塞**（**sti/hlt**）直至收到或从队列被交付。
  * **CHASEROS_SYS_IPC_REPLY（9）**：**RDI** = **WAITING_REPLY** 的对端 PID；**RSI** = **`message_t`**。
  * **CHASEROS_SYS_IPC_CALL（10）**：**RDI** = 目标 PID；**RSI** = 请求 **`message_t`**；**RDX** = 响应 **`message_t`** 输出缓冲区。内核 **Send** 后阻塞直至收到 **Reply**（**sti/hlt**），再写入响应；成功 **RAX=0**。

  **阶段 3**：若内核以 **CHASEROS_HYBRID_FS_VIA_IPC** 构建，**SYS_OPEN** 在用户路径上通过 **PID=3** 服务执行 **`user_fd_open_kpath`**（仍链接同一 VFS；便于后续将服务迁出 ring0）。

4. 参考映射（便于对照）
--------------------------------------------------------------------

  **C（内联汇编）**：见 **user/hello.c**。

  **汇编桩**：**user/chaseros-rt/chaseros_syscalls.asm** — **chaseros_sys_***（含 open / close）。

  **Slime STD**：**import "chaseros/io"**；编号 **chaseros/constants**；errno **chaseros/errno**；返回值解析 **chaseros/syscall**。（**SYS_*** 与 **CHASEROS_SYS_*** 数值一致。）

--------------------------------------------------------------------
5. 版本与兼容
--------------------------------------------------------------------

  变更系统调用编号或寄存器语义时，应递增 ChaserOS 对外说明版本，并保留旧编号一段过渡期（策略待定）。  
  工具链与 STD 应以 **kernel/syscall_abi.h** 与本文档为准进行自动化校验（可选后续工作）。
